# Visual Point-and-Click Selector UI — Design Spec

**Date:** 2026-06-01
**Status:** Approved

---

## Overview

A visual interface for creating scraper monitors without writing CSS selectors manually. The user loads a target URL inside a proxied panel, clicks elements to pin labeled fields, previews the results, and confirms to create the monitor.

---

## Architecture

### New pieces

**A. Proxy Route** — `GET /api/proxy?url=<url>`
- Playwright renders the target URL fully (JS executed, dynamic content hydrated)
- Strips `X-Frame-Options` and `Content-Security-Policy` headers from response
- Injects two scripts before `</body>`: `finder.js` (CSS selector generator) and `interceptor.js` (click handler)
- Returns full HTML, served from our domain → no CORS issues in iframe
- Timeout: 30s

**B. Click Interceptor** — `interceptor.js` (injected into proxied page)
- Hover → blue outline on element (like DevTools)
- Click → prevents default navigation, opens floating label popup
- User types field name → pin registered: `{ label, css }` (css generated by `finder`)
- Communicates pins to parent frame via `window.postMessage`

**C. Preview Endpoint** — `POST /api/preview`
- Input: `{ url: string, selectors: { label: string, css: string }[] }`
- Playwright scrapes the URL, extracts values for each selector
- Generates regex for each extracted value (see Regex Generation below)
- Output: `{ label, css, regex, values: string[], count: number }[]`
- Timeout: 30s

**D. Creation UI** — `/monitors/new`
- Split layout: left panel 40% (config), right panel 60% (iframe with proxied page)
- State managed in React: accumulated pins, form fields, preview results

**E. Home Dashboard** — `/`
- Stats cards: active monitors, alerts today, changes this week
- Table: Name | URL | Status (●green/●yellow/●red) | Last run | Actions (view, pause, delete)
- "New Monitor" button → navigates to `/monitors/new`

### Modified pieces

**`SelectorConfig` type** — extended to include regex:
```ts
// lib/shared/types/monitor.types.ts
export type SelectorConfig = {
  field: string;
  css: string;
  regex: string; // now required, not optional
};
```

**Worker fallback logic** — `worker/index.ts`:
1. Try CSS selector → if 0 results, try regex against full raw HTML
2. If both fail → register `SELECTOR_MISSING` change and notify

## Regex Generation

Pipeline (no LLM, no external cost):

### Step 1: Type inference
Check extracted value against known types:

| Type | Detection | Regex |
|---|---|---|
| Price | starts with `$`, `€`, `£` or ends with currency code | `/[$€£]\s*[\d,]+\.?\d*/` |
| Rating | contains digit + `stars`, `estrellas`, `/5`, `/10` | `/[\d.]+\s*(\/\s*\d+\|stars?\|estrellas?)/i` |
| Count | digit + `reviews`, `reseñas`, `items`, `resultados` | `/[\d,]+\s*(reviews?\|reseñas?\|items?\|resultados?)/i` |
| Availability | matches known availability strings | `/(en stock\|out of stock\|agotado\|available\|unavailable)/i` |
| Percentage | ends with `%` | `/[\d.]+\s*%/` |

### Step 2: Anchor + variable fallback (if no type matched)
1. Split value into tokens: numeric sequences and text sequences
2. Numeric sequences → `[\d,]+\.?\d*`
3. Text tokens adjacent to numbers → literal anchor (escaped)
4. Pure text values → first 3 words as literal anchor + `.*`

Examples:
- `"Quedan 3 unidades"` → `/Quedan\s+[\d,]+\s+unidades/i`
- `"4.5 de 5"` → `/[\d.]+\s+de\s+5/i`
- `"Nike Air Max 90"` → `/Nike Air Max.*/i`

## Error Handling

| Case | Behavior |
|---|---|
| URL fails to load (Playwright timeout) | Banner in iframe: "No se pudo cargar la página. Verifica la URL." |
| Site blocks iframe (X-Frame-Options) | Header stripped server-side by Playwright — not an issue |
| Preview timeout | Modal shows error with retry button |
| Selector 0 matches in preview | Red badge ❌, Crear Monitor disabled |
| Selector 1 match in preview | Yellow warning ⚠️, Crear Monitor still enabled |
| Worker: CSS selector fails, regex succeeds | Continues normally, logs fallback used |
| Worker: both CSS and regex fail | Registers `SELECTOR_MISSING`, notifies users, does not crash |

- Scheduling UI changes (frequency edited post-creation from monitor detail)
